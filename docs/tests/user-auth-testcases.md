# Test Cases: Authentication + Authorization + โปรไฟล์/ลายเซ็น (slug: user-auth)

อ้างอิงแผน: `docs/plans/user-auth.md`

> เอกสารนี้ + test ที่ FAIL จริง (RED) คือสัญญา API ที่ `dev` ต้อง implement ให้ผ่าน
> ทุก contract ที่ "สมมติ" (ไม่ได้ระบุชัดในแผน) จะมีหมายเหตุกำกับไว้ — dev ใช้ชื่อ/รูปแบบตามนี้เพื่อให้ test ผ่านโดยไม่ต้องเดา

---
